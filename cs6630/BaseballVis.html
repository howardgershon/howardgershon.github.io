Final Project Design Notebook
Back to the Drawing Board
Originally I had planned to turn in a completely different project. I mined a huge data set for another class (complete Wikipedia revision history) and I wanted to use it for my visualization. I worked on it for a week and a half and finally gave up on it. The data set was just too large. I will eventually finish that project but I knew that I wouldn't finish it in time. This left me with a little under a week to start this project. This was an incredibly disheartening turn of events.
Concept
Left without a project I wondered what I should consider for a new subject. Baseball is a sport that I have always thought really lent itself to analysis. This combined with the fact that I thought it would be pretty easy to find baseball data led me to find a problem in the baseball domain. I wondered what kind of an effect do the opening and closing pitchers have on the winning percent of a team during any given season. So I decided to do a time series that looked at the performance of the opening and closing pitchers using 4 different statistics as metric. These 4 statistics are a) strikeouts, b) innings played, c)walks, and d) hits allowed. These stats were taken from the season that the pitchers were playing in. I also wanted to look at win/loss margins. This was kind of an interesing addition because it isn't in the same scope as the rest of the visualization. The win/loss margins are examined one season at a time but the rest of the visualization is looking at 20 seasons (1994-2013). I began with a model that was a pretty basic time series visualization.
As you can see it isn't much to look at. The idea of doing a basic time series graph was very unsatifying to me so I turned my attention to how I could spice it up. After perusing the internet for some design ideas about baseball I saw this picture of the
Astrodome. And I strated thinking about what it would be like if I created a visualization that was the shape of a baseball field. This was good for me because I have really been interested in polar shaped graphs lately. And thus was born the following design.

With the design in a good place it was time to find the data.
Gathering the Data
The hardest part about this project was gathering the data. There are some really good baseball reference websites out there. One that I particularly liked was Baseball Reference. I wrote a couple Java programs to mine this website to get the data that I needed. The first set that I needed was the win/loss margins for every game over a twenty year period. It ended up being about 90,000 bits of data. The second set of data that I needed was a set of 4 statistics for each of the opening and closing pitchers for each team for each season. This only ened up being about 6000 pieces of information.

The nice thing about mining these websites is once you have your program worked out on a small set of infromation it is pretty easy to scale it to larger problems.
Execution
This visualization has a key on the side so that the user can navigate through the different teams in the MLB. All one has to do is point a click at a time to run the visualization for that particular team.
The user can change the statistic being viewed by pressing the j key. The statistic is then shown just below the title.
Also around the edge of the graph the user can select the year for the win/loss bargraph.
Given more time I was going to set up some hover tags that told you who the opening and closing pitchers were and the statiscits of a game that you are hovering over on the win/loss bar chart. I also just realized that I didn't label the graph on the axes. I would do that and make them dynamic so that when the user changes statistics the axes labels would change as well.
Conclusion
Looking at the data it is interesting to me the amount it seems that the closing pitcher influences the game. I don't watch much baseball so I thought that the closing pitcher just finisnes up the game for the starting pitcher. However, see below as you can see the closing pitcher is clearly striking out more people than the opeing pitcher. This came as a surpise to me. Further exploration I am sure will yield even more insight in baseball and pitching.
